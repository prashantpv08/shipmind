# ADR 0011 — PostgreSQL data access and prototype migration

- Status: Accepted
- Date: 2026-07-20
- Extends: ADR 0010

## Context

The prototype stores one validated JSON document and uploaded files locally, with a Vercel Blob branch for the hackathon. Commercial Axiom needs tenant-scoped relational integrity, versioned migrations, concurrency protection, retry-safe external operations, and a verifiable path that preserves the existing stable IDs and provenance.

## Decision

1. Use standard PostgreSQL 17 semantics locally and Amazon RDS for PostgreSQL in the planned AWS deployment. No query may depend on a Neon-, Vercel-, or AWS-proprietary database feature.
2. Use Drizzle ORM 0.45 with `node-postgres` 8 for typed schema/query access. Use checked-in SQL generated by Drizzle Kit for forward migrations. Application code uses one bounded `pg` pool per process.
3. Store top-level graph concepts in relational tables: organizations, workspaces, projects, sources, graph versions, knowledge entities, gaps, clarification questions, architecture options, technology recommendations, decisions, documents, approvals, publications, and wireframe revisions.
4. Use JSONB only for nested value objects and arrays whose elements have no independent lifecycle in the current contract. Canonical IDs, ownership, versions, hashes, truth status, timestamps, and foreign keys remain first-class columns.
5. Every customer-scoped row includes `organization_id`. Composite foreign keys include organization scope where one customer-owned record references another.
6. Project mutations use `row_version` compare-and-update checks. A stale write raises `OptimisticConcurrencyError`.
7. Future external side effects use the `outbox_events` and `idempotency_records` tables. Outbox insertion can participate in the same database transaction as the state mutation; workers claim rows with `FOR UPDATE SKIP LOCKED`.
8. Prototype import is blocked unless Zod validation and explicit reference/provenance checks pass. Apply mode runs in one transaction, records the source hash and counts, rejects a non-empty target organization, and is idempotent for the same snapshot.
9. The JSON store remains an explicit rollback adapter during migration. The commercial local profile selects PostgreSQL with `AXIOM_PROJECT_STORE=postgres`; `json` is migration/rollback-only.

## Why this layer

- Drizzle declares a TypeScript schema while retaining reviewable SQL migrations and standard PostgreSQL drivers: [Drizzle PostgreSQL guide](https://orm.drizzle.team/docs/get-started-postgresql) and [migration guide](https://orm.drizzle.team/docs/migrations).
- `node-postgres` provides a bounded pool and requires a single checked-out client for a transaction, behavior respected by Drizzle transactions: [pooling](https://node-postgres.com/features/pooling) and [transactions](https://node-postgres.com/features/transactions).
- AWS warns that connection churn can exhaust RDS PostgreSQL connections, supporting one bounded application pool now and RDS Proxy only when measured concurrency requires it: [RDS PostgreSQL troubleshooting](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/PostgreSQL.InitialTroubleshooting.html).
- The Docker Official Image provides the standard local initialization contract used by Compose: [PostgreSQL Official Image](https://hub.docker.com/_/postgres).

## Reversibility

- Forward migration: `pnpm db:migrate`.
- Foundation rollback SQL: `drizzle/0000_slow_weapon_omega.down.sql`.
- Local reset is restricted in code to localhost databases named `axiom` or beginning with `axiom_test`.
- The source JSON and uploaded files are not deleted by import.
- Do not point `db:reset` or integration tests at shared, staging, or production databases.

## Consequences

- Schema and migration changes require review together.
- The application can be switched project-store-by-project-store without a flag-day rewrite.
- Authentication and request-derived organization scope remain Milestone 2. Milestone 1 uses the explicit local development organization only.
- Source binaries remain in the existing local allowlisted directory until the S3 storage adapter milestone; PostgreSQL owns source metadata and content references.
